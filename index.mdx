Watchexec is a simple, standalone tool that monitors a path and runs a command whenever it detects file modifications. Whether you're running tests on code changes, rebuilding assets, or restarting a development server, watchexec stays out of your way and just works.

## What watchexec does

When you run `watchexec make`, it immediately executes `make`, then watches the current directory. When any file changes, it runs `make` again. It coalesces rapid sequences of file events (like editor swap files) into a single run, loads your `.gitignore` and `.ignore` files automatically, and uses efficient kernel-level APIs so it doesn't burn CPU in a polling loop.

<Steps>
  <Step title="Install watchexec">
    Install via your preferred method — package manager, Cargo, or a pre-built binary:

    ```bash
    brew install watchexec          # macOS
    cargo binstall watchexec-cli    # any platform with cargo
    ```
  </Step>
  <Step title="Run your first watch">
    Watch the current directory and run a command when anything changes:

    ```bash
    watchexec make
    ```
  </Step>
  <Step title="Filter to specific files">
    Use `-e` to watch only certain file extensions, or `-f` / `-i` for glob patterns:

    ```bash
    watchexec -e rs,toml cargo test
    watchexec -e js,css,html npm run build
    ```
  </Step>
  <Step title="Restart long-running processes">
    Use `-r` to kill and restart a server or daemon on each change:

    ```bash
    watchexec -r -e py -- python server.py
    ```
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="Efficient file watching" icon="eye">
    Uses inotify, FSEvents, and ReadDirectoryChangesW — no polling, minimal CPU overhead.
  </Card>
  <Card title="Smart ignore support" icon="filter">
    Automatically reads `.gitignore`, `.ignore`, and other VCS ignore files.
  </Card>
  <Card title="Cross-platform" icon="globe">
    A single static binary runs on Linux, macOS, Windows, and BSD — no runtime dependency.
  </Card>
  <Card title="Process control" icon="sliders">
    Control busy-update behaviour: queue, restart, or send a signal to running processes.
  </Card>
  <Card title="Event emission" icon="satellite-dish">
    Pass changed file paths to child processes via environment variables or JSON on stdin.
  </Card>
  <Card title="Rust library" icon="code">

  </Card>
