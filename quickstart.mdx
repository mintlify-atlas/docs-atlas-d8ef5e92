Watchexec runs a command once at startup, then watches the current directory for changes and runs the command again each time a file is modified. This guide walks through the most common patterns.

<Steps>
  <Step title="Install watchexec">
    Install via your package manager or cargo:

    ```bash
    brew install watchexec          # macOS
    apk add watchexec               # Alpine Linux
    pacman -S watchexec             # Arch Linux
    cargo binstall watchexec-cli    # any platform with cargo
    ```

    See [Installation](/installation) for all options including pre-built binaries and Windows package managers.
  </Step>
  <Step title="Run your first command">
    Watch the current directory and run `make` whenever anything changes:

    ```bash
    watchexec make
    ```

    Watchexec runs `make` immediately, then watches. It reads your `.gitignore` and `.ignore` files automatically, so build artifacts and VCS directories are excluded without configuration.
  </Step>
  <Step title="Filter to specific file types">
    Use `-e` to limit triggering to files with specific extensions:

    ```bash
    watchexec -e js,css,html npm run build
    ```

    Only changes to `.js`, `.css`, or `.html` files will trigger the build. This is the fastest way to narrow the watch scope.
  </Step>
  <Step title="Restart a running process">
    For long-running processes like development servers, use `-r` to kill and restart on each change:

    ```bash
    watchexec -r -e py -- python server.py
    ```

    Watchexec sends `SIGTERM` to the running process, waits up to 10 seconds for a clean exit, then starts a new one.
  </Step>
</Steps>

## Common patterns

<CodeGroup>
  ```bash Watch specific directories
  # Watch only lib/ and src/, ignoring everything else
  watchexec -w src -w lib make
  ```

  ```bash Ignore patterns
  # Skip everything under target/ (e.g. Rust build output)
  watchexec -i "target/**" make test
  ```

  ```bash Run two commands
  # Shell features like semicolons work because commands run in a shell by default
  watchexec 'date; make'
  ```

  ```bash Watch a single file
  # Run bundle install when Gemfile changes
  watchexec -w Gemfile bundle install
  ```
</CodeGroup>

## Restart and signal control

When a new event arrives while the command is already running, you control what happens with `--on-busy-update`:

| Mode | Behaviour |
|---|---|
| `do-nothing` | Ignore events while the command runs (default) |
| `queue` | Queue one more run after the current one finishes |
| `restart` | Kill the command and start a new one (same as `-r`) |
| `signal` | Send a signal without restarting (configure with `--signal`) |

```bash
# Restart with SIGKILL instead of the default SIGTERM
watchexec -r --stop-signal SIGKILL my_server

# Send SIGHUP to reload config, without restarting
watchexec -n --signal SIGHUP my_server
```

## Event emission

Watchexec can pass changed file paths to the child process. The default since 2.0 is `none` — no event information is emitted. Use `--emit-events-to` to enable it:

<CodeGroup>
  ```bash Environment variables (legacy)
  # Sets $WATCHEXEC_WRITTEN_PATH, $WATCHEXEC_CREATED_PATH, etc.
  watchexec --emit-events-to=environment make
  ```

  ```bash JSON on stdin
  # Sends a JSON object per event to the command's stdin
  watchexec --emit-events-to=json-stdio my_script.py
  ```
</CodeGroup>

### Consume events without running a command

Use `--only-emit-events` to get a stream of JSON events and process them yourself:

```bash
watchexec --emit-events-to=json-stdio --only-emit-events
```

Each changed file produces one JSON object on stdout:

```json
{"tags":[{"kind":"source","source":"filesystem"},{"kind":"fs","simple":"modify","full":"Modify(Data(Any))"},{"kind":"path","absolute":"/home/user/project/src/main.rs","filetype":"file"}]}
```

This is useful for building custom tooling on top of watchexec's file-watching engine.

## Timing and diagnostics

```bash
# Print how long each command run takes
watchexec --timings -- make
```

```
[Running: make]
...
[Command was successful, lasted 52.748081074s]
```

```bash
# Show which events are triggering the command (useful for debugging filters)
watchexec --print-events make

# Full diagnostic output
watchexec -vvv make
```

## Desktop notifications

Get a system notification when a command starts and ends:

```bash
watchexec -N go build
```

<Tip>
On macOS, watchexec uses native notification center. On Linux, it uses libnotify if available. On Windows, it uses toast notifications.
</Tip>
